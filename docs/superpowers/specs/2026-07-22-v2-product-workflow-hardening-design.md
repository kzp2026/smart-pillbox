# V2 产品工作流完善设计

## 目标

把 V2 从展示型控制台完善为单用户私有产品决策工作台：修复初始化失败时的恢复体验，清除右下角自定义机器人和 Streamlit Cloud 浮标，确保所有概览、进度和历史严格归属当前产品，并补齐费用保护、生成进度、结果评审、证据追溯、筛选和管理分层。

## 已确认方向

- 保留现有深海军蓝视觉、原站、V2 URL、登录、私有 schema、历史文件和下载能力。
- 当前产品是全站唯一工作上下文；概览、流程进度、设计、Prompt、效果图和历史不得串产品。
- 概览动作和产品行必须是可操作控件，不保留伪按钮卡片。
- 付费图片默认 1 张；确认区显示最大调用张数和费用提示，未确认不得调用。
- 长任务逐阶段写入状态并在页面显示进度；Streamlit Cloud 无独立常驻 worker，因此不承诺关闭页面后继续执行，但已完成阶段和部分图片必须持久化、可重试。
- 结果评审按运行记录保存：决策状态、1–5 分、备注、最终版本标记；同产品可选择两条运行进行对比。
- 导入保留原评论文本，并允许可选映射评分、日期、型号/版本、渠道和用户标签。
- 设置与迁移分组，恢复归档前必须指定产品。
- 数据库不可用时绝不回退公共库；显示脱敏诊断、重试和 Streamlit 管理入口。

## 架构

1. `v2/domain/models.py` 增加产品级快照和运行评审模型。
2. `v2/migrations/002_product_workflow_hardening.sql` 增加运行评审表与评论元数据列；PostgreSQL 初始化顺序执行所有编号 migration，SQLite schema 同步。
3. `v2/adapters/postgres.py` 提供产品级快照、证据列表、评审保存/读取；所有查询限定 `owner_id`。
4. `v2/application/history.py` 把评审和证据组装成页面可消费的数据。
5. `v2/application/image_generation.py` 支持进度回调，逐图持久化部分成果。
6. `v2/app.py` 负责真实按钮、产品选择、费用确认、任务进度、评审对比、历史筛选、归档产品归属和初始化恢复页。
7. `v2/ui/theme.py` 删除机器人资源依赖并隐藏 Streamlit Viewer/Status 浮标；保留键盘焦点和移动端布局。

## 错误与安全

- 页面只显示错误分类和操作建议，不回显连接串、主机、用户、密码或第三方原始异常。
- 删除、迁移、归档恢复和付费生成继续要求显式确认。
- 数据库初始化失败清除失效的运行时缓存项后允许重试，避免错误对象长期驻留。
- 新 migration 只做增量扩展，不改写已上线的 `001`。

## 验收

- 原站冻结测试前后通过。
- 概览无跨产品运行；顶部完成状态随当前产品变化。
- 机器人和 Streamlit 右下角浮标不渲染。
- 付费图片默认 1 张，进度回调可验证。
- 评审可持久化、覆盖更新、按 owner 隔离，并可在历史页对比。
- 全量 V2 测试、Skill 契约检查和浏览器视觉检查通过；线上 Secrets 错误必须单独确认。
